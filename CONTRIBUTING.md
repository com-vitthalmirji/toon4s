# Contributing

Thanks for your interest in contributing to toon4s.

## Before you open a pull request

- Open an issue first for large changes.
- Keep pull requests focused and small.
- Add or update tests for behavior changes.
- Update docs when user behavior changes.

## Local checks

- Run project checks before pushing.
- For Spark integration work, run `sparkIntegration/compile` and relevant tests.

## Style and review

- Follow existing code style and project docs.
- Use clear commit messages.
- Be respectful and practical in review discussions.
